# Copilot Instructions: バージョン管理ルール

## セマンティックバージョニング運用方針
- すべての機能追加・修正・変更は、[セマンティックバージョニング](https://semver.org/lang/ja/)に従い、`major.minor.patch` 形式でバージョンを管理する。
- 変更内容ごとに、必ず `CHANGELOG.md` にバージョンごとの履歴を記載する。
- `Unreleased` セクションは原則使用せず、リリースごとに履歴を追記する。
- バージョンアップ時は、`CHANGELOG.md`・`pyproject.toml` などのバージョン番号を同期させ、Gitタグ（例: `v1.2.0`）を付与する。
- バージョンアップの基準：
  - 後方互換性のない変更 → major
  - 機能追加・互換性のある変更 → minor
  - バグ修正・パッチ → patch

## CHANGELOG.md 記載例
```
## [1.2.0] - 2025-12-28
### Added
- 新機能Aを追加
### Changed
- 既存機能Bの仕様変更
### Fixed
- バグCを修正
```

## 運用ルール
- すべてのPull Request/コミットは、上記ルールに従いバージョン・履歴を管理すること。
- ルール違反のコミット・PRはリジェクト対象とする。
